# Interactive Schema Viewer — Implementation Plan

## Goal
Build a single-page HTML app (`docs/index.html`) for GitHub Pages that lets users visually navigate the data model. Two top-level entry points: **Deal** and **Fund**. Expandable/collapsible tree with clickable cross-references.

## Design

### Layout
- **Left sidebar**: Two root nodes (Deal, Fund) as collapsible trees
  - Expanding an entity shows its properties grouped: scalar fields, nested objects, and relationship links
  - Relationship links (UUID refs like `fund_id -> Fund`) are clickable — clicking navigates to and expands that entity
- **Right panel**: Detail view showing the full schema for the selected entity (properties table, types, descriptions, required fields)
- **Breadcrumb trail**: Shows navigation path so user can backtrack

### Two Root Trees

**Deal tree** (deal-centric view):
```
Deal
├── DealParticipant[] → Party, Fund
├── DealAsset[] (discriminated)
│   ├── FACILITY → Facility → Tranche, RateDefinition, Covenant, AmortizationSchedule
│   ├── INVESTMENT → Investment → PortfolioCompany, Realization
│   ├── PROPERTY → Property → Lease, Tenant, PropertyValuation, NOIStatement, Appraisal
│   └── LP_INTEREST → LPInterest → SecondaryTrade → Transaction → TransferRecord
├── DealMilestone[]
└── Document[]
```

**Fund tree** (fund lifecycle view):
```
Fund
├── FundTerms
├── Commitment[] → SideLetterTerms
├── CapitalCall[] → CallAllocation[]
├── Distribution[] → DistributionAllocation[]
├── NAVStatement → InvestorNAV[], InvestmentValuation[]
├── WaterfallStructure → WaterfallTier[], CarryAllocation
├── Investment[] → PortfolioCompany, Realization
├── Portfolio → Position (Fund/Credit/RE), CashFlow, FundPerformance
└── Party (GP, Fund Admin)
```

### Technical Approach
- **Single `docs/index.html`** — all HTML/CSS/JS inline, zero dependencies
- **Schema data embedded as JSON** — the entity graph is hardcoded from the schema analysis (no runtime schema loading needed for GitHub Pages)
- **CSS**: clean, minimal design. Collapsible tree uses `<details>/<summary>` or JS toggle. Color-coded domains (core=gray, fund=blue, credit=orange, real_estate=green, deal=purple, secondaries=teal, portfolio=indigo, trade_lifecycle=red)
- **JS**: click handler on entity links scrolls/expands target entity. History stack for breadcrumb.

### Entity Data Structure (embedded in JS)
For each entity, embed:
- `name`, `domain`, `schemaFile`
- `properties[]`: name, type, required, description, refEntity (if it links to another entity)
- `isAuditable`: boolean
- `enums`: inline enum values where applicable
